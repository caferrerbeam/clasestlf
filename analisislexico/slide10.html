🍔 Maximal Munch - Regla de la Cadena Más Larga

📖 Definición
Regla de la Cadena Más Larga
Cuando múltiples patrones coinciden con el input desde la misma posición,
siempre seleccionar el lexema más largo posible
📝 En inglés: "Maximal Munch Rule" o "Longest Match Rule"
🎯 Propósito: Resolver ambigüedades cuando el input puede interpretarse de múltiples maneras

🔄 Orden de Aplicación de Reglas

1️⃣ PRIMERO: Maximal Munch

El AFD consume la cadena más larga posible que coincida con el patrón de identificador:

Input: "ifVariable"
AFD: q0 → q1 → q1 → q1 → ... (10 chars)
Lexema: "ifVariable"

2️⃣ SEGUNDO: Tabla de Keywords

Después de obtener el lexema completo, se consulta la tabla:

Lexema: "ifVariable"
keywords.get("ifVariable") → null
Resultado: IDENTIFIER
⚠️ CLAVE: Maximal Munch se aplica ANTES que la tabla de keywords
Por eso "ifVariable" se reconoce como identificador completo, no como "if" + "Variable"
🔄 Múltiples AFDs
⚙️ Implementación
  1. Cada AFD evalúa el input independientemente
  2. Se recolectan TODOS los matches válidos
  3. Se comparan las longitudes de lexemas
  4. Se selecciona el lexema más largo
public class MultipleAFDLexer {
    public Token nextToken(String input, int pos) {
        List<Token> candidates = new ArrayList<>();
        
        // Evaluar TODOS los AFDs
        for (TokenMatcher matcher : matchers) {
            Token token = matcher.tryMatch(input, pos);
            if (token != null) {
                candidates.add(token);
            }
        }
        
        // 🍔 MAXIMAL MUNCH: Elegir el más largo
        return candidates.stream()
            .max(Comparator.comparing(
                token -> token.getLexeme().length()))
            .orElseThrow(() -> 
                new LexicalError("No valid token"));
    }
}
⚠️ Problema: Evalúa AFDs innecesariamente si ya encontró un match largo
⚡ AFD Único Global
🎯 Implementación
  1. Continúa leyendo mientras hay transiciones válidas
  2. Marca el último estado de aceptación alcanzado
  3. Al fallar, retorna el token del último estado válido
  4. Automáticamente garantiza maximal munch
public class SingleAFDLexer {
    public Token nextToken(String input, int pos) {
        State currentState = START;
        State lastAcceptingState = null;
        int lastAcceptingPos = pos;
        int currentPos = pos;
        
        // 🍔 MAXIMAL MUNCH: Continúa mientras pueda
        while (currentPos < input.length()) {
            char c = input.charAt(currentPos);
            State nextState = transition(currentState, c);
            
            if (nextState == null) break; // No hay transición
            
            currentState = nextState;
            currentPos++;
            
            // Marcar si es estado de aceptación
            if (isAccepting(currentState)) {
                lastAcceptingState = currentState;
                lastAcceptingPos = currentPos;
            }
        }
        
        if (lastAcceptingState == null) {
            throw new LexicalError("Invalid token");
        }
        
        String lexeme = input.substring(pos, lastAcceptingPos);
        return createToken(lastAcceptingState, lexeme);
    }
}
✅ Ventaja: Maximal munch automático, sin comparaciones adicionales

🔄 nextToken() devuelve UN SOLO token

⚠️ Importante

nextToken() analiza el input desde la posición actual y devuelve exactamente un token. Para obtener todos los tokens, se debe llamar repetidamente.

→

✅ Solución

Se implementa un método tokenize() que llama a nextToken() en un loop hasta encontrar el token EOF.

💻 Implementación completa del Lexer con Tabla de Símbolos

public class CompleteLexer {
    private String input;
    private int position;
    
    // 📊 TABLA DE SÍMBOLOS - Palabras reservadas
    private static final Map<String, TokenType> SYMBOL_TABLE = 
        Map.of(
            "if",     TokenType.IF_KEYWORD,
            "else",   TokenType.ELSE_KEYWORD,
            "while",  TokenType.WHILE_KEYWORD,
            "for",    TokenType.FOR_KEYWORD,
            "int",    TokenType.INT_TYPE,
            "return", TokenType.RETURN_KEYWORD
        );
    
    public CompleteLexer(String input) {
        this.input = input;
        this.position = 0;
    }
    
    // ⭐ nextToken() devuelve UN SOLO token
    public Token nextToken() throws LexicalException {
        skipWhitespace();
        
        if (position >= input.length()) {
            return new Token(TokenType.EOF, "", position);
        }
        
        // 1️⃣ AFD único procesando desde position actual
        Token rawToken = processSingleAFD();
        
        // 2️⃣ CONSULTAR TABLA DE SÍMBOLOS después de Maximal Munch
        Token finalToken = checkSymbolTable(rawToken);
        
        position += finalToken.getLexeme().length();
        return finalToken;
    }
    
    // 📋 Consultar tabla de símbolos
    private Token checkSymbolTable(Token token) {
        if (token.getType() == TokenType.IDENTIFIER) {
            // ⚠️ DESPUÉS de Maximal Munch, verificar si es palabra reservada
            TokenType keywordType = SYMBOL_TABLE.get(token.getLexeme());
            if (keywordType != null) {
                return new Token(keywordType, token.getLexeme(), token.getPosition());
            }
        }
        return token; // Mantener el token original
    }
    
    // 🔄 tokenize() construye la lista COMPLETA
    public List<Token> tokenize() throws LexicalException {
        List<Token> tokens = new ArrayList<>();
        
        while (true) {
            Token token = nextToken(); // 📍 UNA llamada = UN token
            tokens.add(token);
            
            if (token.getType() == TokenType.EOF) {
                break; // Terminamos cuando encontramos EOF
            }
        }
        
        return tokens;
    }
    
    // Ejemplo de uso mostrando la tabla de símbolos en acción:
    public static void main(String[] args) {
        CompleteLexer lexer = new CompleteLexer("if ifVariable while x");
        List<Token> tokens = lexer.tokenize();
        
        // 🔍 RESULTADO mostrando tabla de símbolos:
        // Token(IF_KEYWORD, "if")        ← Encontrado en tabla
        // Token(IDENTIFIER, "ifVariable") ← NO en tabla (Maximal Munch)
        // Token(WHILE_KEYWORD, "while")   ← Encontrado en tabla  
        // Token(IDENTIFIER, "x")         ← NO en tabla
        // Token(EOF, "")
        
        for (Token token : tokens) {
            System.out.println(token);
        }
    }
}

📊 Tabla de Símbolos - Orden Crítico

❌ ORDEN INCORRECTO

1. Consultar tabla de símbolos
2. Aplicar Maximal Munch

Problema: "ifVariable" se reconocería como "if" + "Variable"

⚠️
✅ ORDEN CORRECTO

1. Aplicar Maximal Munch
2. Consultar tabla de símbolos

Resultado: "ifVariable" se reconoce completo, luego se verifica si es palabra reservada (no lo es)

🎯 Flujo detallado con Tabla de Símbolos para "if ifVariable while":

📍 Llamada 1

1. AFD: "if" → IDENTIFIER
2. Tabla: SYMBOL_TABLE.get("if") = IF_KEYWORD
3. Resultado: Token(IF_KEYWORD, "if")

📍 Llamada 2

1. AFD: "ifVariable" → IDENTIFIER (Maximal Munch!)
2. Tabla: SYMBOL_TABLE.get("ifVariable") = null
3. Resultado: Token(IDENTIFIER, "ifVariable")

📍 Llamada 3

1. AFD: "while" → IDENTIFIER
2. Tabla: SYMBOL_TABLE.get("while") = WHILE_KEYWORD
3. Resultado: Token(WHILE_KEYWORD, "while")

🏆 Lista Final: [IF_KEYWORD, IDENTIFIER, WHILE_KEYWORD, EOF]
💡 Punto Clave: "ifVariable" NO se fragmenta en "if" + "Variable" porque Maximal Munch se aplica ANTES que la tabla de símbolos.

🏆 Conclusión

Maximal Munch es fundamental para lexers deterministas.
nextToken() devuelve un solo token con Maximal Munch garantizado.
Tabla de símbolos se consulta DESPUÉS de Maximal Munch.
tokenize() construye la lista completa llamando nextToken() repetidamente.

🔄 Orden correcto: AFD → Maximal Munch → Tabla de Símbolos → Token Final
