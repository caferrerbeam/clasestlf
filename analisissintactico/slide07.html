📊 Ejemplo Detallado: Parseando "x = 5"

🌱 Top-Down (Recursive Descent)

1. Empezar con Program
2. Program → Statement
3. Statement → Assignment
4. Assignment → ID "=" Expression
5. Consumir tokens: x, =, 5
6. ✅ Árbol completo!
Decisión: ¿Qué regla usar?
Mira el próximo token para decidir

🌳 Bottom-Up (LR)

1. Stack: [] | Input: x = 5
2. Stack: [x] | Input: = 5
3. Stack: [x, =] | Input: 5
4. Stack: [x, =, 5] | Input: []
5. Reducir: x = 5 → Assignment
6. ✅ Árbol completo!
Decisión: ¿Cuándo reducir?
Usa tabla de parsing para decidir

🔧 Pseudocódigo de los Algoritmos

Recursive Descent

function parseExpression() { if (currentToken == ID) { token = advance() if (currentToken == "=") { advance() // consume "=" right = parseExpression() return new Assignment(token, right) } } throw ParseError("Expected assignment") } // Predictivo: decide por lookahead function parseStatement() { switch (currentToken.type) { case IF: return parseIf() case WHILE: return parseWhile() case ID: return parseAssignment() } }

LR Parser

function parseLR() { stack = [0] // estado inicial while (true) { state = stack.top() token = currentToken action = table[state][token] if (action == SHIFT(s)) { stack.push(token, s) advance() } else if (action == REDUCE(rule)) { // Pop elementos de la regla items = stack.pop(rule.length * 2) newState = goto[stack.top()][rule.lhs] stack.push(rule.lhs, newState) } else if (action == ACCEPT) { return stack[1] // AST raíz } else { throw ParseError() } } }

📊 ¿Qué son las Tablas de Parsing?

🎯 Concepto Fundamental

Las tablas de parsing son estructuras de datos que contienen las decisiones precalculadas del parser para cada situación posible durante el análisis sintáctico.

🔑 Diferencias Clave

📋 Tabla LL (Top-Down)
  • Predictiva: decide qué regla expandir
  • Entrada: (No-terminal, Terminal)
  • Salida: regla de producción
  • Tamaño: más pequeña
  • Construcción: relativamente simple
📋 Tabla LR (Bottom-Up)
  • Basada en estados: shift/reduce/goto
  • Entrada: (Estado, Terminal/No-terminal)
  • Salida: acción (shift/reduce/accept)
  • Tamaño: más grande
  • Construcción: compleja (autómatas)

🎯 Nuestro Enfoque en el Curso

Nos enfocaremos en Top-Down (Recursive Descent) porque:
✅ Es más fácil de entender conceptualmente
✅ Permite implementación directa desde la gramática
✅ Genera mejores mensajes de error
✅ Es perfecto para aprender los fundamentos

Bottom-Up es más potente, pero lo mencionaremos solo como referencia
